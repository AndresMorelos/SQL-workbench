¿Qué es SQL?

SQL (Structured Query Language — Lenguaje Estructurado de Consulta) es el idioma estándar que usamos para comunicarnos con bases de datos. Con SQL podemos pedir, agregar, modificar y eliminar información.

Piénsalo así: la base de datos es como una bodega enorme llena de información, y SQL es el idioma con el que le dices al encargado qué información necesitas o qué cambios hacer.

💡 Se pronuncia
"Es-Cu-El" o "Sequel". Ambas formas son correctas.
🌐 Es universal
Funciona en MySQL, PostgreSQL, SQL Server, Oracle, SQLite y muchos más.
📅 Lleva décadas
Se creó en los años 70 y sigue siendo el estándar mundial para manejo de datos.
🗄️
La base de datos

Una base de datos organiza la información en tablas, muy parecidas a hojas de cálculo. Cada tabla representa un tema o entidad del mundo real.

Nuestra base de datos de práctica tiene 4 tablas relacionadas entre sí:

🏢 departamentos
Los departamentos de la empresa: RR.HH., Tecnología, Finanzas…
👤 empleados
Los empleados: nombre, cargo, salario, a qué departamento pertenecen…
📁 proyectos
Los proyectos activos, su presupuesto y estado actual.
📋 asignaciones
Qué empleado trabaja en qué proyecto y con qué rol.

Cada tabla tiene columnas (los tipos de información) y filas (los registros individuales). Por ejemplo, la tabla empleados:

id_empleado nombre cargo salario id_departamento
1AnaGerente RRHH75,0001
2CarlosDesarrollador Senior85,0002
3MaríaAnalista Financiero68,0003
💡 La columna id_departamento en empleados es una clave foránea (FK): conecta cada empleado con su departamento en la tabla departamentos. Así es como las tablas se relacionan entre sí.

Las 4 operaciones CRUD

CRUD es el acrónimo de las 4 operaciones fundamentales que podemos hacer sobre cualquier base de datos. Todo sistema de información del mundo usa estas cuatro acciones.

LetraOperaciónComando SQL¿Qué hace?
C Create — Crear INSERT Agrega filas nuevas a una tabla.
R Read — Leer SELECT Consulta y muestra datos. No modifica nada.
U Update — Actualizar UPDATE Modifica valores en filas ya existentes.
D Delete — Eliminar DELETE Borra filas permanentemente. ¡Sin deshacer!
🔍
SELECT
CONSULTAR DATOS

SELECT es la operación más usada. Sirve para leer y mostrar datos de la base de datos. No modifica nada — es completamente segura.

Sintaxis básica
SELECT columna1, columna2
FROM nombre_tabla
WHERE condicion         -- opcional
ORDER BY columna;      -- opcional
Ejemplo 1 — Traer todas las columnas
SELECT *
FROM empleados;
💡 El símbolo * (asterisco) significa "todas las columnas". Es útil para explorar, pero en sistemas reales es mejor pedir solo las columnas que necesitas.
Ejemplo 2 — Columnas específicas
SELECT nombre, apellido, cargo, salario
FROM empleados;
Ejemplo 3 — Con filtro y ordenamiento
SELECT nombre, salario
FROM empleados
WHERE salario > 70000
ORDER BY salario DESC;
✅ SELECT es seguro: no importa cuántas veces lo ejecutes, los datos en la base de datos no cambian. Úsalo libremente para explorar y verificar información.
➕
INSERT
INSERTAR DATOS

INSERT agrega filas nuevas a una tabla. Debes indicar qué tabla, qué columnas y qué valores vas a insertar.

Sintaxis — especificando columnas (recomendado)
INSERT INTO nombre_tabla (col1, col2, col3)
VALUES (valor1, valor2, valor3);
Sintaxis — sin especificar columnas
INSERT INTO nombre_tabla
VALUES (valor1, valor2, valor3); -- mismo orden que la tabla
Ejemplo — Insertar un nuevo empleado
INSERT INTO empleados
VALUES (
  9, 'Sofía', 'Morales', 's.morales@corp.com',
  'Analista de Datos', 65000, 2, '2025-01-10'
);
💡 Reglas de los valores:
• Los textos van entre comillas simples: 'Ana'
• Los números NO llevan comillas: 75000
• Las fechas van en formato 'AAAA-MM-DD': '2025-01-15'
✏️
UPDATE
ACTUALIZAR DATOS

UPDATE modifica valores en filas que ya existen. Siempre se usa junto con SET (qué cambiar) y casi siempre con WHERE (a qué filas).

Sintaxis
UPDATE nombre_tabla
SET columna1 = nuevo_valor,
    columna2 = otro_valor   -- puedes cambiar varias a la vez
WHERE condicion;             -- ¡SIEMPRE usar WHERE!
Ejemplo 1 — Actualizar el salario de un empleado
UPDATE empleados
SET salario = 72000
WHERE id_empleado = 3;
Ejemplo 2 — Aumento del 10% a todo un departamento
UPDATE empleados
SET salario = salario * 1.10
WHERE id_departamento = 2;
⚠️ ¡Cuidado con UPDATE sin WHERE!
Si escribes UPDATE empleados SET salario = 0 sin WHERE, se actualizarán todos los empleados de la tabla. Siempre verifica tu WHERE antes de ejecutar.
💡 Truco: antes de hacer un UPDATE, ejecuta primero un SELECT con el mismo WHERE para verificar qué filas van a cambiar. Cuando estés seguro, convierte el SELECT en UPDATE.
🗑️
DELETE
ELIMINAR DATOS

DELETE borra filas de una tabla de forma permanente e irreversible. Es la operación que hay que usar con más cuidado.

Sintaxis
DELETE FROM nombre_tabla
WHERE condicion; -- ¡SIEMPRE usar WHERE!
Ejemplo 1 — Eliminar un empleado por ID
DELETE FROM empleados
WHERE id_empleado = 8;
Ejemplo 2 — Eliminar proyectos completados
DELETE FROM proyectos
WHERE estado = 'Completado';
⚠️ DELETE sin WHERE elimina TODO. DELETE FROM empleados borra cada fila de la tabla. No hay botón de deshacer. Siempre usa WHERE.
🔍
WHERE — Filtrar filas

WHERE es la cláusula que filtra qué filas se ven, modifican o eliminan. La base de datos revisa cada fila y solo pasan las que cumplen la condición.

Para combinar varias condiciones usamos AND (y) y OR (o):

AND — se deben cumplir TODAS las condiciones
SELECT * FROM empleados
WHERE salario > 60000
  AND id_departamento = 2;
OR — basta con que se cumpla UNA condición
SELECT * FROM empleados
WHERE nombre = 'Ana'
   OR nombre = 'Carlos';
↕️
ORDER BY — Ordenar resultados

ORDER BY indica en qué orden deben aparecer las filas en el resultado. Sin ORDER BY el orden es impredecible.

↑ ASC (por defecto)
Ordena de menor a mayor: A→Z, 1→100, fechas más antiguas primero.
↓ DESC
Ordena de mayor a menor: Z→A, 100→1, fechas más recientes primero.
Ejemplo — Empleados ordenados por salario (mayor primero)
SELECT nombre, apellido, salario
FROM empleados
ORDER BY salario DESC;
Ejemplo — Ordenar por varias columnas
SELECT apellido, nombre
FROM empleados
ORDER BY apellido ASC, nombre ASC; -- primero por apellido, luego nombre

JOIN — Unir tablas

JOIN combina filas de dos o más tablas basándose en una columna en común. Es la herramienta que nos permite ver información relacionada de varias tablas al mismo tiempo.

Por ejemplo: para ver el nombre del empleado y el nombre de su departamento necesitamos unir las tablas empleados y departamentos.

👤 empleados

id_empleado: 1 | Ana
id_departamento: 1
🔗

🏢 departamentos

id_departamento: 1

Sintaxis
SELECT a.columna, b.columna
FROM tabla_a a              -- alias "a" para tabla_a
JOIN tabla_b b               -- alias "b" para tabla_b
  ON a.columna_comun = b.columna_comun;
Ejemplo — Empleado con su departamento
SELECT e.nombre, e.cargo, d.nombre AS departamento
FROM empleados e
JOIN departamentos d
  ON e.id_departamento = d.id_departamento;
💡 ¿Para qué sirve el alias? Cuando usas dos tablas, puede haber columnas con el mismo nombre (por ejemplo, nombre en empleados y en departamentos). El alias (e.nombre, d.nombre) indica de qué tabla viene cada columna.

Funciones de cálculo

SQL incluye funciones que realizan cálculos sobre un conjunto de filas y devuelven un solo valor como resultado. Son muy usadas en reportes y análisis.

COUNT(*)
Cuenta cuántas filas hay en el resultado.
¿Cuántos empleados tenemos?
SUM(col)
Suma todos los valores numéricos de una columna.
Total de salarios.
AVG(col)
Calcula el promedio de los valores.
Salario promedio.
MIN(col) / MAX(col)
Encuentra el valor más bajo o más alto.
Salario mínimo y máximo.
Ejemplo — Varias funciones a la vez
SELECT
  COUNT(*) AS total_empleados,
  AVG(salario) AS salario_promedio,
  MAX(salario) AS salario_maximo,
  MIN(salario) AS salario_minimo
FROM empleados;
💡 AS sirve para darle un nombre legible a la columna en el resultado. Sin AS, el resultado mostraría COUNT(*) como nombre de columna.
✅
Buenas prácticas

Estas reglas te ayudarán a escribir SQL más seguro, más claro y con menos errores:

  • Siempre usa WHERE en UPDATE y DELETE. Sin WHERE afectas todas las filas. Antes de ejecutar, revisa dos veces tu condición.
  • Prueba con SELECT antes de modificar. Si vas a hacer un UPDATE o DELETE, primero haz un SELECT con el mismo WHERE para ver exactamente qué filas cambiarán.
  • Escribe las palabras clave en MAYÚSCULAS. SELECT, FROM, WHERE, INSERT… en mayúsculas. Las tablas y columnas en minúsculas. No es obligatorio, pero es la convención universal y hace el código más legible.
  • Usa IDs para identificar filas. Prefiere WHERE id_empleado = 3 sobre WHERE nombre = 'Ana'. Los IDs son únicos; los nombres pueden repetirse.
  • Indenta tu código. Pon cada cláusula en su propia línea. Es más fácil de leer y revisar.
  • Termina con punto y coma (;). Es la forma correcta de cerrar una consulta SQL.
  • Ten cuidado con DELETE y UPDATE masivos. En un sistema real, una operación sin WHERE puede borrar o modificar millones de registros en segundos.
